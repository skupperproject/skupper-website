Using Skupper with Google Kubernetes Engine

Developer tools

The Google Cloud SDK provides a command-line tool called gcloud for using the features of Google Cloud, including Google Kubernetes Engine (GKE).

See the Google Cloud SDK getting started guide for more information.

Logging in

Use the gcloud command and the Google Cloud console to log in:

  1. Go to console.cloud.google.com and sign in.
  2. Select Kubernetes Engine in the menu on the left.
  3. Navigate to your cluster.
  4. Select the Connect link at the top of the page.
  5. Copy the provided gcloud command.
  6. Paste the gcloud command into your console session.

See Configuring cluster access for kubectl for more information.

Enabling outbound connections

By default, private GKE clusters do not allow outbound connections to external IP addresses. To link a private GKE cluster to a remote site with Skupper, you need to enable outbound connections using Cloud NAT.

See the Cloud NAT overview for more information.
